01 🧭 Que Es Este Proyecto

Este proyecto es una tienda virtual de ropa deportiva con un asistente conversacional integrado. En palabras simples: la persona puede navegar productos como en un e-commerce normal y, al mismo tiempo, hablar con un bot que entiende pedidos como "busco zapatillas negras" o "cual es el precio de este modelo".

La solucion junta tres partes grandes que trabajan en equipo: una interfaz web para ver productos, una API en Python para procesar mensajes, y un motor de inteligencia artificial para detectar la intencion del usuario. Ademas, el sistema guarda memoria de la conversacion para no perder el contexto entre mensajes.

1) Interfaz visual (Frontend)

Muestra catalogo, filtros, carrito y chat. Es la parte que ve la persona en el navegador.

2) Logica del asistente (Backend)

Recibe cada mensaje, interpreta que quiere el usuario y devuelve una respuesta util y accionable.

3) Datos y modelo entrenado

Incluye productos, intenciones de conversacion y el modelo de IA listo para usar.

Idea clave en lenguaje simple: imagina que este proyecto es una tienda con un vendedor virtual. El vendedor escucha lo que pides, entiende tu necesidad, aplica filtros en el catalogo y te devuelve opciones concretas sin que tengas que conocer comandos tecnicos.

02 🚀 Puesta En Marcha Paso A Paso

# 1) Instalar dependencias del proyecto
pip install -r requirements.txt

# 2) Iniciar servidor (API + Frontend en el mismo puerto)
python app.py

# 3) Abrir en el navegador
http://localhost:5000/
Nota importante: El archivo data/modelo_lstm.pth ya viene entrenado. Solo debes ejecutar entrenar_modelo_lstm.py si cambiaste data/intenciones_chatbot.json y quieres que el bot aprenda esos cambios.
Si algo falla al iniciar:
  • Revisa que exista la carpeta data con intenciones_chatbot.json, modelo_lstm.pth y products_scraped.json.
  • Verifica que el puerto 5000 no este siendo usado por otro programa.
  • Si spaCy pide modelo de idioma, el propio proyecto intentara descargarlo automaticamente.

03 📁 Arquitectura De Archivos Completa

Aqui tienes la estructura completa del proyecto.

CHATBOT/ ├── app.py ├── catalogo.py ├── config.py ├── bot/motor_dialogo.py ├── extractor.py ├── ia.py ├── memoria.py ├── entrenar_modelo_lstm.py ├── utils_nlp.py ├── interfaz_chatbot.html ├── requirements.txt ├── Historial_Tecnico.md ├── css/ │ └── estilos_principales.css ├── js/ │ └── interfaz_principal.js ├── data/ │ ├── intenciones_chatbot.json │ ├── modelo_lstm.pth │ ├── products_scraped.json │ └── chat_memory └── documentacion/ └── interfaz_chatbot.html
Lectura rapida del grafico: el frontend no decide por su cuenta la logica del bot. Siempre pasa por app.py y bot/motor_dialogo.py, y desde ahi se consulta IA, catalogo y memoria para construir una respuesta coherente.

Bloque 1: Backend y logica del asistente (Python)

app.py

Puerta principal del sistema. Crea el servidor Flask y publica rutas como /chat, /search, /products, /transcribe y /status. En terminos simples, aqui entra todo lo que llega desde el navegador y aqui tambien sale la respuesta final. Tambien sirve los archivos web para que no necesites otro servidor adicional.

bot/motor_dialogo.py

Cerebro de decisiones conversacionales. Toma la intencion detectada por la IA y decide que hacer: responder, buscar productos, pedir mas detalle o usar contexto de mensajes anteriores. Tambien construye mensajes naturales para que el bot suene mas humano y aplica reglas para evitar respuestas incoherentes.

ia.py

Motor de inferencia. Carga el modelo de PyTorch desde data/modelo_lstm.pth y calcula que intencion tiene el mensaje del usuario. Adicionalmente inicializa el modelo de voz Whisper para transcribir audios enviados desde el frontend.

catalogo.py

Gestor de inventario y busqueda. Carga productos desde data/products_scraped.json, normaliza categorias y construye un indice para buscar rapido por nombre, descripcion, color, talla, genero o precio. Usa TF-IDF y similitud de coseno, con fallback lexico, para entregar resultados mas utiles.

extractor.py

Extractor de entidades. Detecta datos concretos dentro de frases libres: color, talla, presupuesto, genero y tambien posibles IDs de producto. Combina regex, normalizacion de texto y coincidencia difusa con thefuzz para tolerar errores de escritura.

memoria.py

Memoria conversacional. Guarda estado por sesion (ultimo tag, filtros, producto seleccionado e historial corto). Gracias a esto el usuario puede decir "y de color rojo" despues de otra busqueda y el bot entiende que sigue en el mismo tema.

config.py

Parametros globales. Centraliza rutas de archivos y umbrales de comportamiento (confianza, limites de busqueda, memoria, etc.). Tener todo en un solo lugar evita cambios sueltos y reduce errores de mantenimiento.

utils_nlp.py

Utilidades de lenguaje. Limpia texto, elimina acentos y genera tokens lematizados con spaCy. Tambien crea bigramas para conservar significado en frases como "zapatilla blanca" o "entrega rapida".

entrenar_modelo_lstm.py

Entrenamiento del modelo. Prepara datos de intents, crea vocabulario, entrena una red neuronal con Embedding + LSTM bidireccional, evalua metricas y guarda el modelo final. Se usa cuando quieres mejorar o actualizar el entendimiento del bot.

Bloque 2: Frontend y experiencia del usuario (Web)

interfaz_chatbot.html (raiz del proyecto)

Estructura visual de la tienda. Define layout general: panel de catalogo, filtros, chat, carrito y controles de busqueda. Es la base de la interfaz que ve el usuario final.

css/estilos_principales.css

Estilos y diseno responsive. Da apariencia moderna al dashboard, define tarjetas de producto, animaciones, panel de chat y version movil. Sin este archivo la web funcionaria, pero se veria sin formato y dificil de usar.

js/interfaz_principal.js

Comportamiento dinamico de la web. Carga productos, aplica filtros, sincroniza carrito, envia mensajes al endpoint /chat, procesa filter_action del backend y maneja la grabacion de audio para transcripcion. Es el puente principal entre interfaz y API.

Bloque 3: Datos, modelo entrenado y persistencia

data/intenciones_chatbot.json

Base de conocimiento conversacional. Contiene intenciones (tags), ejemplos de frases (patterns) y posibles respuestas. Es la materia prima para entrenar la IA y tambien una referencia de comportamiento del bot.

data/modelo_lstm.pth

Modelo ya entrenado. Guarda pesos y metadatos de la red neuronal. Gracias a este archivo el bot puede arrancar en segundos sin volver a entrenar cada vez.

data/products_scraped.json

Catalogo de productos. Lista de articulos con campos como id, nombre, precio, categoria, colores, tallas, genero y stock. Es la fuente principal para mostrar el inventario y para responder consultas del usuario.

data/chat_memory

Memoria volátil. Anteriormente se usaba shelve para disco. Ahora se almacena en memoria RAM y un Session ID dinámico en el navegador limpia el historial en cada recarga (F5) para prevenir cruce de filtros ("zombies").

Bloque 4: Soporte, instalacion y documentacion

requirements.txt

Lista de librerias. Define todo lo necesario para ejecutar la aplicacion (Flask, torch, spaCy, scikit-learn, thefuzz, faster-whisper, entre otros). Evita instalar paquetes manualmente uno por uno.

documentacion/interfaz_chatbot.html

Manual tecnico del proyecto. Este mismo archivo. Su funcion es explicar arquitectura, flujo, endpoints y tecnicas en un lenguaje entendible para personas nuevas en estos temas.

Historial_Tecnico.md

Bitacora de cambios. Registra mejoras, hotfixes, pruebas y decisiones tecnicas para mantener trazabilidad. Es muy util cuando el proyecto crece y varias personas lo mantienen.

04 ⚙️ Flujo Completo De Mensajeria

Cuando una persona escribe en el chat o usa el microfono, el mensaje no se responde de forma magica. Recorre una cadena de pasos bien definidos para mantener precision y contexto.

Etapa 1: Captura desde la interfaz

El frontend (js/interfaz_principal.js) envia el texto al endpoint /chat junto con session_id y, si aplica, context_product_id. Esto permite que el backend sepa quien habla y sobre que producto se esta consultando.

Etapa 2: Prediccion de intencion (ia.py)

La IA convierte el texto en numeros y calcula la etiqueta mas probable (por ejemplo: buscar_producto, consultar_precio_item, pedidos). Tambien retorna nivel de confianza para que la capa de dialogo decida si confiar totalmente o apoyar con reglas.

Etapa 3: Extraccion de filtros y contexto

bot/motor_dialogo.py llama a extractor.py para detectar categoria, color, talla, genero y precio maximo. Luego consulta memoria.py para heredar contexto cuando corresponde. Si la frase depende de un producto previamente seleccionado, se prioriza ese contexto.

Etapa 4: Busqueda en catalogo

Si la intencion es de busqueda, catalogo.py filtra y ordena productos usando reglas duras (color, talla, precio) y similitud de texto. Cuando no hay coincidencia semantica fuerte, intenta una via lexico-practica para no dejar al usuario sin opciones.

Etapa 5: Respuesta para UI

El backend devuelve texto natural + tag + filter_action. Con filter_action, el frontend ajusta automaticamente botones, sliders y tarjetas visibles. Resultado: la conversacion y el catalogo quedan sincronizados.

05 🧠 Motor De IA Explicado Facil

El bot no usa un modelo generativo gigante. Usa un clasificador entrenado para identificar intenciones concretas del negocio. Eso lo hace mas rapido y estable para un entorno de tienda.

En este proyecto, la red real es de tipo Embedding + LSTM bidireccional + capa final de clasificacion. En simple: transforma palabras en vectores, analiza secuencia de la frase y escoge la intencion mas probable.

Como interpretarlo sin tecnicismos:

  • Entrada: El texto se limpia y se transforma en una secuencia numerica.
  • Embedding: Cada palabra se convierte en una representacion matematica que conserva significado.
  • LSTM bidireccional: Analiza orden y contexto de palabras, leyendo la secuencia en ambos sentidos.
  • Salida: Se calcula la probabilidad de cada intencion y se elige la mejor.
Resumen sencillo: la IA no "inventa" respuestas largas. Primero clasifica la intencion y luego la logica del proyecto arma una respuesta util segun reglas y datos del catalogo.

06 🧪 Tecnicas De Procesamiento De Lenguaje

Limpieza + Lematizacion

utils_nlp.py normaliza tildes y caracteres, luego spaCy convierte palabras a su forma base. Asi "buscando zapatillas" y "buscar zapatilla" quedan mas cerca y el bot entiende mejor.

Extraccion de Entidades

extractor.py detecta color, talla, genero y precio usando regex y thefuzz. Esto permite que frases informales como "quiero algo negro talla m" se conviertan en filtros concretos.

Busqueda Semantica + Lexica

catalogo.py combina TF-IDF con similitud de coseno y, si hace falta, una busqueda por coincidencia textual. Asi se reduce el riesgo de "no encontre nada" cuando si hay productos parecidos.

Estas tecnicas se eligieron porque son estables, rapidas y adecuadas para una tienda. No requieren infraestructura compleja y funcionan bien con un catalogo en JSON y un modelo de intenciones.

07 🔌 API Del Proyecto En Palabras Simples

Un endpoint es una "puerta" del servidor. El frontend toca esa puerta y recibe una respuesta. Este proyecto tiene endpoints claros para cada tarea:

Ruta Metodo Para que sirve
/ GET Entrega la pagina principal de la tienda (interfaz_chatbot.html).
/status GET Muestra estado del servidor, catalogo activo y modelo cargado.
/products GET Devuelve lista de productos para renderizar el catalogo.
/search POST Permite buscar con filtros estructurados (categoria, color, genero, precio, keywords).
/chat POST Procesa mensajes del chat y devuelve respuesta + acciones de filtro para UI.
/transcribe POST Convierte audio en texto con Whisper para alimentar el chat.
/save_pdf POST Guarda en el servidor la boleta de venta PDF generada en el frontend.
Por que este grafico importa: muestra que la UI no solo recibe un texto. Tambien recibe instrucciones estructuradas (filter_action) para mover filtros y mostrar exactamente los productos que el bot encontro.

08 🛠️ Glosario De Funciones Clave

Estas funciones son las mas importantes para entender y mantener el proyecto. Estan explicadas con enfoque practico para que puedas ubicar rapido donde modificar cuando necesites mejorar el comportamiento del sistema.

Predecir_Tag(Texto_Consulta) [ia.py]

Recibe texto del usuario, lo transforma al formato que entiende el modelo y devuelve la etiqueta con mayor probabilidad. Tambien retorna confianza y margen para que la logica decida si necesita apoyo de heuristicas.

Obtener_Respuesta_Principal(...) [bot/motor_dialogo.py]

Funcion central de negocio. Une IA, extractor, memoria y catalogo para generar la respuesta final. Maneja continuidad de contexto, forzado de intenciones por reglas y armado de filter_action para sincronizar catalogo y chat.

Buscar_Productos(...) [catalogo.py]

Aplica filtros directos (categoria, color, talla, genero y precio) y luego ordena por relevancia. Si no hay match semantico suficiente, usa coincidencia lexico-textual para rescatar resultados utiles.

Extraer_Filtros(...) [extractor.py]

Interpreta frases libres y detecta entidades concretas con tolerancia a errores de escritura. Devuelve categoria, color, precio maximo, talla y genero para que la busqueda sea precisa.

Actualizar_Contexto(...) [memoria.py]

Guarda estado por sesion en disco (shelve): ultimo tag, filtros usados y producto activo. Esto permite preguntas encadenadas como "y su precio?" sin repetir todo.

tokenizar_y_lematizar(texto) [utils_nlp.py]

Limpia texto, aplica lematizacion y genera bigramas. Mejora la consistencia del lenguaje para IA y busqueda semantica.

Detectar_Id_De_Producto_En_Texto(...) [extractor.py]

Detecta cuando el usuario menciona un producto concreto por nombre o codigo. Esto habilita respuestas de detalle (precio, stock, colores) sobre el item correcto.

chat() [app.py]

Es el endpoint principal del asistente. Recibe JSON, coordina contexto, llama la logica de dialogo y devuelve respuesta estructurada para el frontend.

Aplicar_Filtro_Desde_Chat(...) [js/interfaz_principal.js]

Toma la accion que devuelve el backend y actualiza la interfaz: categoria activa, color, genero, precio y productos visibles. Es clave para que el usuario vea reflejada la conversacion en el catalogo.

Obtener_Respuesta_Chat(...) [js/interfaz_principal.js]

Envia el mensaje al backend, muestra indicador de escritura y dibuja la respuesta del bot en pantalla. Tambien conserva o limpia contexto de producto segun el tag recibido.

Consejo para personas nuevas: si quieres modificar el comportamiento del bot, empieza por bot/motor_dialogo.py. Si quieres cambiar visual y experiencia, empieza por interfaz_chatbot.html + css/estilos_principales.css + js/interfaz_principal.js. Si quieres mejorar entendimiento del lenguaje, revisa intenciones_chatbot.json y entrenar_modelo_lstm.py.
