Guía Explicativa: Sistema de Chatbot SENATI Sports

1. ¿De qué trata este proyecto?

Este sistema es un Chatbot Inteligente E-commerce diseñado para vender artículos deportivos. Permite a los usuarios interactuar mediante texto y voz, hacer consultas sobre productos (precio, stock, color), agregar productos a un carrito virtual y finalizar la compra generando una boleta electrónica (PDF) mientras se actualiza automáticamente el inventario en la base de datos.

2. Estructura de Archivos y Módulos (¿Para qué sirve cada cosa?)

El código está ordenado en diferentes carpetas ("módulos") para no tener todo mezclado. Cada carpeta tiene una responsabilidad única:

  • app.py: Es el corazón y punto de entrada. Recibe las peticiones de internet (desde el navegador), decide a qué módulo llamar y devuelve la respuesta. Es el "director de orquesta".
  • bot/ (El Cerebro del Chatbot):
    • motor_dialogo.py: Decide qué responder basado en lo que el usuario dice.
    • inteligencia_artificial.py: Carga el modelo matemático entrenado (Red Neuronal LSTM) para clasificar textos.
    • catalogo_productos.py: Se encarga de buscar productos, filtrar por precio/color y administrar el stock visible.
    • memoria_conversacion.py: Permite que el bot "recuerde" de qué estaban hablando. (Ej. Si preguntas "precio" luego de ver una zapatilla, sabe que hablas de esa zapatilla).
  • core/ (La Base del Sistema):
    • base_datos.py: Habla directamente con MySQL para guardar ventas y leer productos.
    • configuracion.py: Contiene variables globales (como contraseñas, URLs y límites).
  • admin/ (Panel de Control): Permite a los dueños de la tienda ver las ventas, editar productos y administrar el negocio sin tocar el código.
  • Archivos Web (interfaz_chatbot.html, css/, js/): Son la "Cara" de la aplicación. Lo que el usuario ve, los botones que presiona y los estilos visuales.

3. Flujo Principal: ¿Qué pasa cuando un usuario envía un mensaje de Chat?

Imagina que el usuario escribe "Quiero unas zapatillas Nike baratas". Esto es lo que ocurre internamente, paso a paso:

Paso 1: El Navegador habla con el Servidor.
El archivo JavaScript (js/interfaz_principal.js) toma el texto del usuario y lo envía a la ruta (Endpoint) /chat en app.py mediante una petición POST.
Paso 2: Comprobación de la Memoria (Contexto).
El servidor pregunta: "¿Este usuario estaba hablando de algún producto específico antes?". Si acababa de hacer clic en una zapatilla y escribe "precio", el sistema devuelve el precio directamente sin pasar por toda la inteligencia artificial compleja.
Paso 3: Análisis de Intención (Inteligencia Artificial).
Si no es un mensaje rápido o directo, se le entrega al motor_dialogo.py, el cual usa el modelo entrenado para darse cuenta de que la intención es "buscar_zapatillas" y la acción es "filtrar_por_marca_nike" y "ordenar_precio_menor".
Paso 4: Devolver la Respuesta.
El servidor junta un texto amigable (Ej. "¡Claro! Aquí tienes nuestras Nike más económicas") junto con las órdenes de filtrado, y lo empaca en un formato JSON. Se lo regresa al navegador.
Paso 5: El Navegador actualiza la Pantalla.
El JavaScript recibe el JSON, pinta el mensaje del bot en la conversación y hace que el catálogo de productos de la derecha se mueva para mostrar solo los zapatos pedidos.

4. Flujo de Transcripción de Voz: ¿Cómo funciona el audio?

Cuando el usuario no quiere escribir y decide enviar un mensaje de voz presionando el botón del micrófono:

  • El navegador utiliza la API nativa de reconocimiento de voz (SpeechRecognition o webkitSpeechRecognition) para transcribir el audio directamente en el cliente.
  • El texto reconocido se inserta automáticamente en la caja de chat como si el usuario lo hubiera tecleado.
  • Si el navegador no soporta reconocimiento de voz, se muestra un mensaje indicando que debe escribir el mensaje manualmente.

5. Flujo de Compra: ¿Qué pasa cuando el usuario paga?

Este es el momento más crítico para cualquier tienda. El proceso es el siguiente:

Paso 1: Presionar "Comprar / Generar Boleta".
El usuario añade cosas al carrito y le da a comprar. El carrito completo (una lista con qué producto y cuántas unidades) se envía a /generate_pdf en el app.py.
Paso 2: Generación del Documento (PDF).
El servidor usa una librería de Python (como reportlab) para dibujar un archivo PDF desde cero, insertando el título "Boleta de Venta SENATI SPORTS", calculando los subtotales, totales, fecha y dibujando líneas.
Paso 3: Guardado Seguro en la Base de Datos.
Antes de entregar el PDF, el sistema hace dos cosas urgentes en MySQL:
a. Inserta un nuevo registro en la tabla ventas y su venta_detalle.
b. Ejecuta un comando UPDATE productos SET stock = stock - cantidad comprada para asegurarse de que nadie más pueda comprar algo que ya se agotó.
Paso 4: Entrega de la Boleta.
El servidor toma el PDF recién creado y se lo envía al navegador para que el usuario se lo descargue como recibo de su transacción.

En resumen: Todo este proyecto es un puente que conecta una interfaz visual brillante y amigable con una Base de Datos rígida y segura, usando Inteligencia Artificial para entender a los humanos en el medio de los dos.
